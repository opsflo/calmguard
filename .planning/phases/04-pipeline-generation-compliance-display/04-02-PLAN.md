    - Read `analysisResult.risk.nodeRiskMap` from Zustand — array of `{ nodeId: string, nodeName: string, riskLevel: 'critical' | 'high' | 'medium' | 'low', riskFactors: string[], complianceGaps: number }` (NOTE: `complianceGaps` is an INTEGER COUNT, not a string array).
    - Read `analysisResult.risk.frameworkScores` from Zustand — array of `{ framework: string, score: number, rating: string }`.
    - Read `analysisResult.compliance.frameworkMappings` from Zustand — FLAT array where each element is one control row: `{ framework: 'SOX' | 'PCI-DSS' | 'CCC' | 'NIST-CSF', controlId: string, controlName: string, calmControlId: string | null, status: 'compliant' | 'partial' | 'non-compliant' | 'not-applicable', evidence: string, recommendation: string, severity: string }`. This is NOT nested by framework — it's a flat list.
    - Read `status` from Zustand.

    **Heat map grid structure:**
    - Columns: 4 frameworks (SOX, PCI-DSS, NIST-CSF, CCC) — use `frameworkScores` to determine which frameworks have results.
    - Rows: each node from `nodeRiskMap`.
    - Cell value for `[node][framework]`: derive from the node's riskLevel and complianceGaps count. Simple mapping:
      - If node's `complianceGaps > 0` AND node's `riskLevel` is 'high' or 'critical' → 'non-compliant' (red)
      - If node's `complianceGaps > 0` AND node's `riskLevel` is 'medium' → 'partial' (amber)
      - If node's `complianceGaps === 0` AND node's `riskLevel` is 'low' → 'compliant' (emerald)
      - If node's `riskLevel` is 'medium' → 'partial' (amber)
      - If node's `riskLevel` is 'high' or 'critical' → 'non-compliant' (red)
      - If node's `riskLevel` is 'low' → 'compliant' (emerald)
      - Fallback for no data: 'not-applicable' (slate-700)
    - NOTE: Since `complianceGaps` is a count (not per-framework), the same cell status applies across all frameworks for a given node. This is acceptable for the hackathon — the heat map shows risk posture per node, not per-framework-per-node granularity.

    **Cell colors:**
    ```typescript
    const cellColors = {
      compliant: 'bg-emerald-500/30 border-emerald-500/20',
      partial: 'bg-amber-500/30 border-amber-500/20',
      'non-compliant': 'bg-red-500/30 border-red-500/20',
      'not-applicable': 'bg-slate-700/30 border-slate-700/20',
      loading: 'bg-slate-700/50 border-slate-700/30',
    };
    ```

    **Transition effect:**
    - Initialize all cells as `'loading'` state. When `analysisResult` becomes available, update each cell to its actual status.
    - Apply `transition-colors duration-700 ease-in-out` to each cell div for the fade-from-gray effect.

    **Tooltip:**
    - Import `Tooltip`, `TooltipContent`, `TooltipProvider`, `TooltipTrigger` from `@/components/ui/tooltip`.
    - Wrap each cell in a `<Tooltip>`:
      ```tsx

        <Tooltip>
          <TooltipTrigger asChild>
            <div className={cn('h-8 rounded border', cellColors[cellStatus], 'transition-colors duration-700 ease-in-out cursor-pointer')} />
          </TooltipTrigger>
          <TooltipContent className="bg-slate-800 border-slate-700 text-slate-200">
            <p className="font-medium">{nodeName}</p>
            <p className="text-xs text-slate-400">{framework}: {cellStatus}</p>
            {findingSummary && <p className="text-xs text-slate-400 mt-1">{findingSummary}</p>}
          </TooltipContent>
        </Tooltip>
      </TooltipProvider>
      ```

    **Layout:**
    ```

      <div className="p-6">
        <h3 className="text-sm font-medium text-slate-400 mb-4">Risk Heat Map</h3>
        {/* Column headers */}
        <div className="grid" style={{ gridTemplateColumns: `180px repeat(${frameworkCount}, 1fr)` }}>
          <div /> {/* empty corner */}
          {frameworks.map(fw => <div key={fw} className="text-xs text-slate-400 text-center font-medium px-2 py-1">{fw}</div>)}
          {/* Rows */}
          {nodes.map(node => (
            <>
              <div className="text-xs text-slate-300 truncate py-1 pr-3">{node.nodeName}</div>
              {frameworks.map(fw => <HeatMapCell key={...} ... />)}
            </>
          ))}
        </div>
      </div>
    </Card>
    ```
