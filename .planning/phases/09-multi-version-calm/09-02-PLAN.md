## Goal

Wire the detected CALM version through the Zustand store to the dashboard header badge. Update all `setCalmData` call sites (7 total) to pass the version. Add the CALM version badge next to the existing "Parsed: N nodes, M relationships" badge. Add the v1.0 API Gateway demo to the architecture selector. After this plan, users see "CALM v1.1" (or v1.0/v1.2) on the dashboard after parsing.

## Context

- **CONTEXT.md locked decisions:** Show version as badge next to parsed info. No special treatment for older versions (no warnings, no amber colors). Badge text: "CALM v1.1".
- **setCalmData call sites** (found via `rg "setCalmData"`): `analysis-store.ts` (definition), `header.tsx`, `calm-upload-zone.tsx`, `architecture-selector.tsx`, `github-input.tsx`, `page.tsx`. Plus API routes that return data consumed by `setCalmData`: `fetch-calm/route.ts`, `calm/parse/route.ts`.
- **Plan 01 output:** `ParseSuccess` now has `version: CalmVersion`. `parseCalm()` returns version. `CalmVersion = '1.0' | '1.1' | '1.2'`.
- **API routes:** `fetch-calm/route.ts` calls `parseCalm()` and returns `{ calm, analysisInput, fileSha, defaultBranch }`. Must add `version` to response. `page.tsx` and `github-input.tsx` consume this response and call `setCalmData`.
- **Store:** `setCalmData: (data: CalmDocument, input: AnalysisInput) => void`. Must become `(data, input, version) => void`.

<task id="09-02-01" title="Add calmVersion to store and update all setCalmData call sites" autonomy="full">

### Objective
Add `calmVersion: CalmVersion | null` to the Zustand store, update `setCalmData` signature to accept version as third parameter, and update all 6 call sites. Also update the 2 API routes to include version in their JSON responses.

### Implementation Details

**analysis-store.ts:**
```typescript
import type { CalmVersion } from '@/lib/calm/normalizer';

interface AnalysisState {
  // ... existing fields ...
  calmVersion: CalmVersion | null; // NEW

  // Actions — updated signature
  setCalmData: (data: CalmDocument, input: AnalysisInput, version?: CalmVersion) => void;
}

const initialState = {
  // ... existing fields ...
  calmVersion: null as CalmVersion | null, // NEW
};

// Updated action
setCalmData: (data, input, version) =>
  set({
    rawCalmData: data,
    analysisInput: input,
    calmVersion: version ?? null,
    status: 'parsed',
    error: null,
  }),

// reset: also clear calmVersion (already handled by initialState spread)
```

IMPORTANT: Make `version` optional (`version?: CalmVersion`) so that call sites that get data from API routes (where version is a JSON string, not a TypeScript type) can pass it, and so that the TypeScript compiler doesn't break all call sites simultaneously. The API route call sites (`page.tsx`, `github-input.tsx`) pass the version from the API response JSON.

**Call site updates (6 files):**

1. **`header.tsx`** (line 44-45): `setCalmData(result.data, input)` -> `setCalmData(result.data, input, result.version)`
2. **`calm-upload-zone.tsx`** (line 88): `setCalmData(localResult.data, input)` -> `setCalmData(localResult.data, input, localResult.version)`
3. **`architecture-selector.tsx`** (line 71): `setCalmData(result.data, analysisInput)` -> `setCalmData(result.data, analysisInput, result.version)`
4. **`github-input.tsx`** (line 103): `setCalmData(successData.calm, successData.analysisInput)` -> `setCalmData(successData.calm, successData.analysisInput, successData.version)`
5. **`page.tsx`** (line 45): `setCalmData(data.calm, data.analysisInput)` -> `setCalmData(data.calm, data.analysisInput, data.version)`

The `github-input.tsx` and `page.tsx` call sites consume data from the `fetch-calm` API route, which needs to include `version` in its response.

**API route updates (2 files):**
