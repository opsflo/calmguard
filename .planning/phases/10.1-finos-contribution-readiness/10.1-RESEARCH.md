## Summary

This phase is entirely about codebase hygiene and reviewer package assembly — not new feature development. The work splits into two streams: code fixes (Items 1-4) and documentation artifacts (Items 5-7). All seven items have clear, bounded scope with no external API dependencies.

The globalThis refactor is the highest-risk item technically: there are exactly 5 globalThis variables (`__lastPipelineResult`, `__lastCloudInfraResult`, `__lastAnalysisResult`, `__lastCalmDocument`, `__agentEventEmitter`) spread across 4 files. The first four are session state written by `/api/analyze` and read by `/api/github/create-pr` and `/api/pipeline`. The fifth (`__agentEventEmitter`) is a hot-reload singleton that intentionally survives webpack restarts and should be treated differently. The session store needs a Map keyed by requestId with TTL cleanup — straightforward in-memory pattern.

The React component testing item requires one important setup fix: `vitest.config.mts` currently only includes `src/__tests__/**/*.test.ts` (no `.tsx`). Component tests will need either a pattern extension or placement in the existing test tree with `.test.tsx` extension added to the include glob. `@testing-library/react` is not present in `package.json` — it must be installed. `jsdom` is already a devDependency and Vitest is already configured with `environment: 'jsdom'`.

**Primary recommendation:** Execute in this order — Item 1 (script, 30 min), Items 5-7 (docs, 2 hrs), Item 3 (CONTRIBUTING.md, 1 hr), Item 4 (tests, 3 hrs), Item 2 (session store, last because highest risk). This sequences scriptable/low-risk items first, saves the architectural refactor for last when all docs are done.

## Architecture Patterns

### Pattern 1: Session Store for Cross-Route State

**What:** Replace globalThis variables with a Map-based session store keyed by requestId UUID.

**Interface:**
```typescript
// src/lib/session-store.ts
interface SessionData {
  pipeline: PipelineConfig | null;
  cloudInfra: CloudInfraConfig | null;
  analysisResult: AnalysisResult | null;
  calmDocument: CalmDocument | null;
  createdAt: number;
}

const sessions = new Map<string, SessionData>();

export function createSession(id: string): void { ... }
export function setSessionData(id: string, partial: Partial<SessionData>): void { ... }
export function getSessionData(id: string): SessionData | undefined { ... }
export function deleteSession(id: string): void { ... }
export function cleanupExpiredSessions(maxAgeMs = 30 * 60 * 1000): void { ... }
```

**Request flow:**
1. `/api/analyze` POST generates `sessionId = crypto.randomUUID()`
2. Passes sessionId through to orchestrator (or writes to store directly after runAnalysis returns)
3. Returns sessionId in SSE `done` event or response header
4. Client stores sessionId in Zustand store
5. `/api/github/create-pr` POST receives sessionId in request body
6. `/api/pipeline` GET receives sessionId in query param

**Cleanup strategy:** Delete session explicitly after create-pr completes successfully. Add TTL cleanup (30 min) as fallback. The `cleanupExpiredSessions` can be called at the start of the analyze POST handler.

### Pattern 2: License Header Script

**Bash approach (confirmed for this project):**

```bash
#!/usr/bin/env bash
# scripts/add-license-headers.sh

TS_HEADER="// SPDX-License-Identifier: Apache-2.0
// Copyright 2026 FINOS"

YAML_HEADER="# SPDX-License-Identifier: Apache-2.0
# Copyright 2026 FINOS"

MD_HEADER="<!-- SPDX-License-Identifier: Apache-2.0 -->
<!-- Copyright 2026 FINOS -->"

add_header() {
  local file="$1"
  local header="$2"
  # Skip if already has SPDX header
  if grep -q "SPDX-License-Identifier" "$file"; then
    return
  fi
  local tmp=$(mktemp)
  printf '%s\n\n' "$header" | cat - "$file" > "$tmp" && mv "$tmp" "$file"
}

# TypeScript / TSX
while IFS= read -r -d '' file; do
  add_header "$file" "$TS_HEADER"
done < <(find src -name "*.ts" -o -name "*.tsx" -print0)

# YAML (agents/)
while IFS= read -r -d '' file; do
  add_header "$file" "$YAML_HEADER"
done < <(find agents -name "*.yaml" -print0)

# Markdown (skills/)
while IFS= read -r -d '' file; do
  add_header "$file" "$MD_HEADER"
done < <(find skills -name "*.md" -print0)
```

**Edge case for Markdown:** Skills files may have YAML front matter (they don't currently — confirmed 8 files are plain Markdown). HTML comment approach is cleanest and does not require front matter insertion.

## Common Pitfalls

### Pitfall 1: Breaking the `__agentEventEmitter` globalThis
**What goes wrong:** Replacing ALL globalThis references with session store breaks the event emitter singleton, which is intentionally global (not session-scoped).
**Why it happens:** Grep for `globalThis` finds 5 results; naive replacement hits the emitter.
**How to avoid:** The emitter in `src/lib/ai/streaming.ts` must remain a globalThis singleton. Only replace `__lastPipelineResult`, `__lastCloudInfraResult`, `__lastAnalysisResult`, `__lastCalmDocument`.
**Warning signs:** SSE streaming stops working (events never reach the client).

### Pitfall 2: Vitest include pattern excludes .tsx tests
**What goes wrong:** Component tests with `.test.tsx` extension are not discovered and appear to "pass" (0 tests collected).
**Why it happens:** `vitest.config.mts` currently only includes `src/__tests__/**/*.test.ts`.
**How to avoid:** Update include to `['src/__tests__/**/*.test.{ts,tsx}']` before writing component tests.
**Warning signs:** `pnpm test:run` reports the same test count as before adding component tests.

### Pitfall 3: requestAnimationFrame in jsdom
**What goes wrong:** `ComplianceCard` calls `requestAnimationFrame` via the `useCountUp` hook. jsdom does not implement rAF, causing "requestAnimationFrame is not a function" errors.
**Why it happens:** jsdom is a DOM simulation environment, not a browser.
**How to avoid:** Add `global.requestAnimationFrame = vi.fn((cb) => { cb(0); return 0; })` and `global.cancelAnimationFrame = vi.fn()` in test setup or per-test file.

### Pitfall 4: Double-write to `__lastCloudInfraResult`
**What goes wrong:** `orchestrator.ts:266` writes `globalThis.__lastCloudInfraResult = cloudInfra` AND `analyze/route.ts:94` also writes it. When moving to session store, if the orchestrator write is kept, it requires passing the sessionId into `runAnalysis()`, coupling the library to the HTTP layer.
**Why it happens:** The orchestrator write was added before the analyze route write existed.
**How to avoid:** Remove the write from `orchestrator.ts` entirely. The `analyze/route.ts` write after `runAnalysis()` returns is the single source of truth.

### Pitfall 5: CONTRIBUTING.md is already good — don't overwrite it
**What goes wrong:** CONTEXT.md says "current state: 3 sentences (hackathon template)" but the actual file is 148 lines with full sections.
**Why it happens:** The file was expanded between the audit and today.
**How to avoid:** Do NOT replace CONTRIBUTING.md — only ADD the "Extending CALMGuard" section. Merge the fork workflow from `docs/docs/contributing.md`.

### Pitfall 6: Docusaurus sidebar won't show adrs/ without config
**What goes wrong:** Creating `docs/docs/adrs/` with `.md` files may not appear in the sidebar automatically.
**Why it happens:** Docusaurus sidebar is configured in `docs/sidebars.ts`. Auto-discovery depends on sidebar config mode.
**How to avoid:** Check `docs/sidebars.ts` and add an ADR category entry if needed. With auto-generated sidebars this should work, but verify.

## Code Examples

### Session Store (complete pattern)

```typescript
// src/lib/session-store.ts
// SPDX-License-Identifier: Apache-2.0
// Copyright 2026 FINOS

import type { PipelineConfig } from '@/lib/agents/pipeline-generator';
import type { AnalysisResult } from '@/lib/agents/orchestrator';
import type { CalmDocument } from '@/lib/calm/types';
import type { CloudInfraConfig } from '@/lib/agents/cloud-infra-generator';

interface SessionData {
  pipeline: PipelineConfig | null;
  cloudInfra: CloudInfraConfig | null;
  analysisResult: AnalysisResult | null;
  calmDocument: CalmDocument | null;
  createdAt: number;
}

const sessions = new Map<string, SessionData>();

const DEFAULT_SESSION: Omit<SessionData, 'createdAt'> = {
  pipeline: null,
  cloudInfra: null,
  analysisResult: null,
  calmDocument: null,
};

export function createSession(id: string): void {
  sessions.set(id, { ...DEFAULT_SESSION, createdAt: Date.now() });
}

export function setSessionData(id: string, partial: Partial<Omit<SessionData, 'createdAt'>>): void {
  const existing = sessions.get(id);
  if (!existing) return;
  sessions.set(id, { ...existing, ...partial });
}

export function getSessionData(id: string): SessionData | undefined {
  return sessions.get(id);
}

export function deleteSession(id: string): void {
  sessions.delete(id);
}

export function cleanupExpiredSessions(maxAgeMs = 30 * 60 * 1000): void {
  const now = Date.now();
  for (const [id, data] of sessions.entries()) {
    if (now - data.createdAt > maxAgeMs) {
      sessions.delete(id);
    }
  }
}

export function getSessionCount(): number {
  return sessions.size;
}
```
