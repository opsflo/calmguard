## Goal

Refactor the GitOps card from a 2-column layout to a 3-button horizontal row with icons, add infrastructure PR state to the Zustand store, and implement the concurrency lock (disable all three buttons when any one is generating). After this plan, the dashboard shows three PR buttons — DevSecOps CI, Compliance Remediation, and Cloud Infrastructure — with correct disable/enable behavior and icons. The Cloud Infrastructure button calls the create-pr API route (which will be extended in Plan 10-03).

## Context

- **CONTEXT.md locked decisions:** Three side-by-side buttons, horizontal row, equal width. Icons: shield for DevSecOps, file-check for Compliance, cloud for Infra. Disable all three while any is generating — spinner on active, others greyed out. After success, clicked button becomes link, others re-enable. Buttons only appear after analysis completes.
- **Research:** Pattern 2 (Zustand store extension), Pattern 3 (GitOpsCard 3-button row). Pitfall 4: PRSection `isComingSoon` logic conflicts with disable-all — add explicit `disabled` prop instead of relying on `onGenerate === undefined`. Pitfall 3: Use `grid-cols-1 md:grid-cols-3` with `truncate` on branch names.
- **Current codebase:** `PRRecord.type` is `'pipeline' | 'remediation'`. Store has `pipelinePR` and `remediationPR`. GitOpsCard uses `grid-cols-1 md:grid-cols-2` with `PRSection` sub-component. PRSection uses `onGenerate === undefined` for "Coming Soon" detection.
- **Key pitfall (Pitfall 4):** Don't overload `onGenerate === undefined` for both "coming soon" and "temporarily disabled". Add a `disabled` prop to PRSection.

<task id="10-02-01" title="Extend PRRecord type and Zustand store with infraPR" autonomy="full">

### Objective
Add `'infra'` to the `PRRecord` type union and add `infraPR` state with `setInfraPR` action to the Zustand store.

### Implementation Details

**`src/lib/github/types.ts`** — Extend PRRecord type:

Change the `type` field from `'pipeline' | 'remediation'` to `'pipeline' | 'remediation' | 'infra'`:

```typescript
export interface PRRecord {
  type: 'pipeline' | 'remediation' | 'infra';  // was: 'pipeline' | 'remediation'
  status: 'idle' | 'generating' | 'open' | 'error';
  step?: string;
  branchName?: string;
  prUrl?: string;
  prNumber?: number;
  fileCount?: number;
  error?: string;
}
```

**`src/store/analysis-store.ts`** — Add infraPR state and action:

1. Add to `AnalysisState` interface:
   ```typescript
   infraPR: PRRecord;
   setInfraPR: (pr: Partial<PRRecord>) => void;
   ```

2. Add to `initialState`:
   ```typescript
   infraPR: { type: 'infra' as const, status: 'idle' as const },
   ```

3. Add to `create()` store:
   ```typescript
   setInfraPR: (pr) =>
     set((state) => ({
       infraPR: { ...state.infraPR, ...pr },
     })),
   ```

4. Update `reset()` — `initialState` already contains `infraPR` so no explicit change needed since `reset: () => set(initialState)`.

### Verification
- `pnpm typecheck` passes — `PRRecord` accepts `type: 'infra'`, store has `infraPR` and `setInfraPR`
- Existing `pipelinePR` and `remediationPR` state unaffected
- `initialState.infraPR.type === 'infra'` and `initialState.infraPR.status === 'idle'`

</task>

<task id="10-02-02" title="Refactor GitOpsCard to three-button layout with concurrency lock" autonomy="full">

### Objective
Refactor the GitOpsCard component from a 2-column grid to a 3-button horizontal row with icons (Shield, FileCheck2, Cloud), implement the concurrency lock (disable all buttons when any is generating), and add the Cloud Infrastructure PR handler.

### Implementation Details

**PRSection changes** — Add `disabled` prop and `icon` prop:

```typescript
interface PRSectionProps {
  record: PRRecord;
  label: string;
  description: string;
  icon: React.ReactNode;                    // NEW — icon element
  onGenerate: () => void;                   // CHANGE — always a function, never undefined
  disabled: boolean;                        // NEW — externally controlled disable
}
```

Update PRSection logic:
- Remove `isComingSoon` — no longer needed (all three buttons are real)
- In `idle` state: render the generate button, but add `disabled={disabled}` prop. When disabled, use muted styling (`bg-slate-700 text-slate-400 cursor-not-allowed`) instead of emerald
- In `error` state: show retry button, also respect `disabled` prop
- Add the `icon` to the header section alongside `FileCode` (replace `FileCode` with the passed icon)

**GitOpsCard layout changes:**

1. Import new icons from lucide-react: `Shield`, `FileCheck2`, `Cloud`
2. Read `infraPR` and `setInfraPR` from store
3. Derive `isAnyGenerating`:
   ```typescript
   const isAnyGenerating =
     pipelinePR.status === 'generating' ||
     remediationPR.status === 'generating' ||
     infraPR.status === 'generating';
   ```
4. Add `handleGenerateInfraPR` — same pattern as `handleGeneratePipelinePR` but with `type: 'infra'`:
   ```typescript
   const handleGenerateInfraPR = async () => {
     setInfraPR({ status: 'generating', step: 'Starting...' });
     try {
       const res = await fetch('/api/github/create-pr', {
         method: 'POST',
         headers: { 'Content-Type': 'application/json' },
         body: JSON.stringify({
           type: 'infra',
           owner: githubRepo.owner,
           repo: githubRepo.repo,
           filePath: githubRepo.filePath,
           fileSha: githubRepo.fileSha,
           defaultBranch: githubRepo.defaultBranch,
         }),
       });
       await readPRStream(res, setInfraPR);
     } catch (error) {
       setInfraPR({
         status: 'error',
         error: error instanceof Error ? error.message : 'Failed to generate infrastructure PR',
         step: undefined,
       });
     }
   };
   ```
5. Change grid from `md:grid-cols-2` to `md:grid-cols-3`:
   ```tsx
   <div className="grid grid-cols-1 md:grid-cols-3 gap-4">
     <PRSection
       record={pipelinePR}
       label="DevSecOps CI"
       description="GitHub Actions: lint, build, test, SAST, secrets scan"
       icon={<Shield className="h-4 w-4 text-slate-400 flex-shrink-0" />}
       onGenerate={handleGeneratePipelinePR}
       disabled={isAnyGenerating}
     />
     <PRSection
       record={remediationPR}
       label="Compliance Remediation"
       description="Missing controls added, weak protocols upgraded"
       icon={<FileCheck2 className="h-4 w-4 text-slate-400 flex-shrink-0" />}
       onGenerate={handleGenerateRemediationPR}
       disabled={isAnyGenerating}
     />
     <PRSection
       record={infraPR}
       label="Cloud Infrastructure"
       description="AWS Terraform: VPC, ECS, RDS, security groups"
       icon={<Cloud className="h-4 w-4 text-slate-400 flex-shrink-0" />}
       onGenerate={handleGenerateInfraPR}
       disabled={isAnyGenerating}
     />
   </div>
   ```
6. Update label from "Pipeline Artifacts" to "DevSecOps CI" per locked decisions
7. Update description from "GitHub Actions, SAST configs, IaC templates" to "GitHub Actions: lint, build, test, SAST, secrets scan" (IaC is now in Cloud Infra)

**Important:** When a button's own record has `status === 'generating'`, `PRSection` renders the spinner (existing behavior). The `disabled` prop only affects the idle/error states of the OTHER two buttons. To avoid disabling the currently-generating button's own spinner render, `disabled` is only checked in the idle and error branches — the generating branch renders unconditionally.
