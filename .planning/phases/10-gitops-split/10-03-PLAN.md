## Goal

Extend the create-pr API route to handle three PR types with correct branch naming, PR labels, and content separation. The pipeline PR no longer includes IaC (moved to Cloud Infra PR). The Cloud Infra PR commits Terraform modules from `__lastCloudInfraResult`. All three PR types get auto-labels. Branch names change to the locked decision format.

## Context

- **CONTEXT.md locked decisions:**
  - Branch names: `calmguard/devsecops-ci-{ts}`, `calmguard/compliance-remediation-{ts}`, `calmguard/cloud-infra-{ts}`
  - Auto-label: DevSecOps CI → `ci/cd`, Compliance → `compliance`, Cloud Infra → `infrastructure`. Labels created if they don't exist.
  - PR descriptions contain CALM traceability summary
  - On failure: re-enable buttons with error toast. Failed button shows subtle error indicator.
- **Research:** Pattern 4 (API route extension), Pattern 6 (label creation via GitHub API — 422 means already exists), Pitfall 6 (DevSecOps CI PR still contains IaC — must remove it). `buildPipelineFiles()` currently includes `terraform/main.tf` — this must be removed. Anti-pattern: don't throw on label failures (cosmetic).
- **Current codebase:** `create-pr/route.ts` handles `type: 'pipeline'` and `type: 'remediation'` with SSE streaming. Branch names are `calmguard/pipeline-{ts}` and `calmguard/remediation-{ts}`. `buildPipelineFiles()` includes IaC files (terraform/main.tf or cloudformation/template.yaml).
- **Dependencies:** Plan 10-01 provides `CloudInfraConfig` type and `__lastCloudInfraResult` global. Plan 10-02 provides `type: 'infra'` in the PRRecord union and the client-side handler calling the route with `type: 'infra'`.

**`src/lib/github/operations.ts`** — Add two new exported functions:

```typescript
/**
 * Create a label in a repository, ignoring 422 (already exists).
 * Labels are cosmetic — failure is non-critical.
 */
export async function ensureLabel(
  owner: string,
  repo: string,
  name: string,
  color: string,
  description: string,
  token: string,
): Promise<void> {
  const res = await githubFetch(`/repos/${owner}/${repo}/labels`, {
    method: 'POST',
    token,
    body: JSON.stringify({ name, color, description }),
  });
  // 201 = created, 422 = already exists — both are fine
  if (!res.ok && res.status !== 422) {
    console.warn(`Failed to ensure label "${name}" (${res.status})`);
  }
}

/**
 * Add labels to a PR (PRs are issues in GitHub's data model).
 * Label failures are non-critical — log but don't throw.
 */
export async function addLabelToPR(
  owner: string,
  repo: string,
  prNumber: number,
  labels: string[],
  token: string,
): Promise<void> {
  const res = await githubFetch(`/repos/${owner}/${repo}/issues/${prNumber}/labels`, {
    method: 'POST',
    token,
    body: JSON.stringify({ labels }),
  });
  if (!res.ok) {
    console.warn(`Failed to add labels to PR #${prNumber}: ${res.status}`);
  }
}
```

**New `buildInfraFiles()` function:**

```typescript
function buildInfraFiles(
  infra: CloudInfraConfig,
): Array<{ path: string; content: string }> {
  return infra.terraform.modules.map((mod) => ({
    path: mod.filename,
    content: mod.content,
  }));
}
```

**New `buildInfraPRBody()` function:**
```typescript
function buildInfraPRBody(infra: CloudInfraConfig): string {
  const traceabilityList = infra.traceability
    .map((t) => `- **${t.calmElement}** → \`${t.generatedResource}\`: ${t.rationale}`)
    .join('\n');

  const moduleList = infra.terraform.modules
    .map((m) => `- \`${m.filename}\` — ${m.calmSignal}`)
    .join('\n');

  return [
    '## CALMGuard Cloud Infrastructure',
    '',
    'Generated by [CALMGuard](https://github.com/finos-labs/dtcch-2026-opsflow-llc) compliance analysis.',
    '',
    '### Summary',
    infra.summary,
    '',
    '### Terraform Modules',
    moduleList,
    '',
    '### CALM Traceability',
    '',
    traceabilityList,
    '',
    '---',
    '*Generated by CALMGuard CALM-native compliance platform*',
  ].join('\n');
}
```

```typescript
} else if (type === 'infra') {
  // ----------------------------------------------------------------
  // Cloud Infrastructure PR: multi-file commit (Terraform modules)
  // ----------------------------------------------------------------

  // Step 1: Get HEAD SHA
  emit({ type: 'step', step: 'Getting HEAD SHA...' });
  const headSha = await getHeadSha(owner, repo, defaultBranch, token).catch(/* same error mapping */);

  // Step 2: Create branch

  emit({ type: 'step', step: 'Creating branch...' });
  await createBranch(owner, repo, infraBranchName, headSha, token).catch(/* same error mapping */);

  const infra = globalThis.__lastCloudInfraResult;

    throw new Error('Cloud infrastructure analysis not available. Run analysis first.');
  }

  const infraFiles = buildInfraFiles(infra);
  await commitMultipleFiles(
    owner, repo, infraBranchName, infraFiles,
    'chore: add CALMGuard cloud infrastructure scaffolds',
    headSha, token,
  );

  // Step 4: Open pull request
  emit({ type: 'step', step: 'Opening pull request...' });
  const infraPRBody = buildInfraPRBody(infra);
  const { html_url: infraPrUrl, number: infraPrNumber } = await createPR(
    owner, repo, infraBranchName, defaultBranch,
    'chore: add CALMGuard cloud infrastructure',
    infraPRBody, token,
  );

  // Step 5: Auto-label
  const label = LABEL_MAP[type];
  await ensureLabel(owner, repo, label.name, label.color, label.description, token);
  await addLabelToPR(owner, repo, infraPrNumber, [label.name], token);

  emit({
    type: 'done',
    prUrl: infraPrUrl,
    prNumber: infraPrNumber,
    branchName: infraBranchName,
    fileCount: infraFiles.length,
  });
}
```

## must_haves

These are derived from the phase goal ("Users generate three distinct PRs targeting different review audiences") working backward across all four requirements:

1. **DevSecOps CI PR has NO IaC** — `buildPipelineFiles()` excludes terraform/main.tf and cloudformation/template.yaml; the PR contains only GitHub Actions workflow and security scanning configs [GOPS-02]
2. **Cloud Infra PR has ONLY Terraform** — `buildInfraFiles()` produces files exclusively from `CloudInfraConfig.terraform.modules`; no CI workflows, no CALM modifications [GOPS-03]
3. **Compliance Remediation PR unchanged** — The remediation branch logic, CALM file commit, and PR body generation are identical to v1.2 except for branch name prefix change [GOPS-04]
4. **Branch names match locked decisions** — `calmguard/devsecops-ci-{ts}`, `calmguard/compliance-remediation-{ts}`, `calmguard/cloud-infra-{ts}` [GOPS-01]
5. **Auto-labels applied** — Each PR type gets its label (`ci/cd`, `compliance`, `infrastructure`) created-if-missing and applied [GOPS-01]
6. **Label failures non-blocking** — `ensureLabel()` and `addLabelToPR()` log warnings but never throw, so PR creation succeeds even if label API fails [GOPS-01]
