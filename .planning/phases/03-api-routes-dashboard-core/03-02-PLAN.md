    - Call `startAnalysis()` to reset store state

    - `fetch('/api/analyze', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ calm: calmData }), signal: abortRef.current.signal })`
    - Check response.ok — if not, read error body, set streamStatus to 'error', call setStatus('error'), return

    - In while(true) loop: `const { done, value } = await reader.read()`
      - If done, break
      - Append decoded value to buffer
      - Split buffer on `\n\n` (SSE frame boundary)
      - Last element goes back to buffer (may be incomplete)
      - For each complete frame: if starts with `data: `, parse JSON after `data: `
      - Dispatch parsed event:
        - If `type === 'done'`: call `setAnalysisResult(json.result)`, set streamStatus to 'complete'
        - If `type === 'error'`: call `setStatus('error')`, set streamStatus to 'error'
        - Otherwise: call `addAgentEvent(json)` (it's an AgentEvent)
    - After while loop exits: if streamStatus is still 'running', set to 'complete'
    - Catch block: if err.name !== 'AbortError', set error status
    - Wrap startStream in useCallback with stable deps

    Auto-reconnect logic (per API-05):
    - Maintain a `retryCountRef` (useRef<number>(0)), max 3 attempts
    - In the catch block, when the error is a network/fetch failure (NOT AbortError, NOT a 4xx/5xx response error):
      - If retryCountRef.current < 3:
        - Increment retryCountRef.current
        - Wait with exponential backoff: `await new Promise(r => setTimeout(r, 1000 * 2 ** retryCountRef.current))`
        - Re-invoke startStream(calmData) recursively
        - Return (don't set error status yet)
      - If retryCountRef.current >= 3: set streamStatus to 'error', call setStatus('error')
    - On successful stream connection (after response.ok): reset retryCountRef.current = 0
    - AbortError: never retry (user-initiated abort)
    - HTTP errors (response.ok === false): never retry (server rejected request)

    - Call abortRef.current?.abort()
    - Reset retryCountRef.current = 0
    - Set streamStatus to 'idle'
    - Wrap in useCallback

    ```typescript
    /**
     * Derive per-agent status from agentEvents array.
     * Used by Sidebar to show colored dots (gray/blue/green/red).
     */
    export function getAgentStatus(
      agentEvents: AgentEvent[],
      activeAgents: string[],
      agentName: string
    ): 'idle' | 'running' | 'complete' | 'error' {
      // Check if agent has any events
      const agentEvts = agentEvents.filter(e => e.agent.name === agentName);
      if (agentEvts.length === 0) return 'idle';

      // Check latest event type for this agent
      const lastEvent = agentEvts[agentEvts.length - 1];
      if (lastEvent.type === 'completed') return 'complete';
      if (lastEvent.type === 'error') return 'error';

      // If agent is in activeAgents, it's running
      if (activeAgents.includes(agentName)) return 'running';

      return 'idle';
    }
    ```

    Also export the list of known agent names as a constant (for sidebar to iterate over):
    ```typescript
    export const AGENT_NAMES = [
      'architecture-analyzer',

      'risk-scorer': 'Risk Scorer',

    ```
