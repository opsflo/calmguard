  <name>Task 2: Create Zustand analysis store</name>

  <action>
    Create `src/store/analysis-store.ts` with Zustand using TypeScript:

    ```typescript
    import { create } from 'zustand';
    import type { CalmDocument } from '@/lib/calm/types';
    import type { AnalysisInput } from '@/lib/calm/extractor';

    type AnalysisStatus = 'idle' | 'loading' | 'parsed' | 'analyzing' | 'complete' | 'error';

    interface AnalysisState {
      // Data
      selectedDemoId: string | null;
      rawCalmData: CalmDocument | null;
      analysisInput: AnalysisInput | null;

      // Status
      status: AnalysisStatus;
      error: string | null;

      // Actions
      setSelectedDemo: (demoId: string) => void;
      setCalmData: (data: CalmDocument, input: AnalysisInput) => void;
      setStatus: (status: AnalysisStatus) => void;
      setError: (error: string) => void;
      reset: () => void;
    }
    ```

    Implement the store with `create<AnalysisState>()`:
    - `setSelectedDemo`: sets demoId, resets previous data
    - `setCalmData`: sets rawCalmData + analysisInput, status to 'parsed'
    - `setStatus`: updates status
    - `setError`: sets error message and status to 'error'
    - `reset`: clears all state back to initial values

    Export the hook and the type:
    ```typescript
    export { useAnalysisStore };
    export type { AnalysisState, AnalysisStatus };
    ```

    **IMPORTANT:**
    - This is a `'use client'` compatible module (Zustand uses React hooks)
    - Do NOT add `'use client'` directive to the store file itself — it's a library module imported by client components
    - Status field supports future phases: 'analyzing' and 'complete' will be used when agents run
    - Keep the store flat (no nested objects for state) — Zustand works best with flat state
  </action>

    - Store exports `useAnalysisStore` hook
    - Store has all state fields: selectedDemoId, rawCalmData, analysisInput, status, error
    - All actions work: setSelectedDemo, setCalmData, setStatus, setError, reset
    - No `any` types
  </verify>
