  <action>
    Create `src/lib/calm/types.ts` with comprehensive Zod schemas mirroring the CALM v1.1 specification.

    **Node types enum** (all 9 types):
    `actor`, `ecosystem`, `system`, `service`, `database`, `network`, `ldap`, `webclient`, `data-asset`

    **Protocol enum** (all 12 protocols):
    `HTTP`, `HTTPS`, `FTP`, `SFTP`, `JDBC`, `WebSocket`, `SocketIO`, `LDAP`, `AMQP`, `TLS`, `mTLS`, `TCP`

    **Schema hierarchy (build bottom-up):**
    1. `interfaceDefinitionSchema` ã `{ unique-id, definition-url?, config? }`
    2. `controlRequirementSchema` ã `{ requirement-url, config-url?, config? }`
    3. `controlDefinitionSchema` ã `{ description, requirements[]? }`
    4. `calmNodeSchema` ã required: `unique-id`, `node-type`, `name`, `description`; optional: `interfaces[]`, `controls{}`, `metadata`, `details`
    5. Relationship schemas using discriminated union on `relationship-type`:
       - `interacts`: `{ actor: string, nodes: string[] }`
       - `connects`: `{ source: { node, interfaces[] }, destination: { node, interfaces[] } }`
       - `deployed-in`: `{ container: string, nodes: string[] }`
       - `composed-of`: `{ container: string, nodes: string[] }`
       - `options`: `z.any()` (structure TBD per research open question)
    6. `flowTransitionSchema` ã `{ relationship-unique-id, sequence-number (int), description?, direction? }`
    7. `calmFlowSchema` ã required: `unique-id`, `name`, `description`, `transitions[]`
    8. `calmDocumentSchema` ã `{ nodes[], relationships[], controls{}?, flows[]? }`

    **Export both Zod schemas AND inferred TypeScript types:**
    ```typescript
    export const calmNodeSchema = z.object({ ... });
    export type CalmNode = z.infer<typeof calmNodeSchema>;
    ```

    **IMPORTANT constraints:**
    - Use `.min(1)` on required string fields with descriptive error messages
    - Use `.optional()` for all optional CALM fields
    - Use `z.discriminatedUnion('relationship-type', [...])` for relationships
    - Controls key pattern: `z.record(z.string(), controlDefinitionSchema)` (skip regex validation for simplicity)
    - Do NOT use `any` ã use `z.unknown()` only for truly opaque fields
    - Add JSDoc comments on each schema explaining what CALM entity it represents
    - Add a `CALM_SCHEMA_VERSION = '1.1'` constant at the top

<task type="auto">
  <name>Task 2: Build CALM parser and data extractor</name>
  <files>
    src/lib/calm/parser.ts

  <action>
    **Parser (`src/lib/calm/parser.ts`):**

    Create discriminated union result type:
    ```typescript
    interface ParseSuccess { success: true; data: CalmDocument }
    interface ParseError { success: false; error: { message: string; issues: Array<{ path: string; message: string; code: string }> } }
    type ParseResult = ParseSuccess | ParseError;
    ```

    Implement `parseCalm(json: unknown): ParseResult`:
    - Use `calmDocumentSchema.safeParse(json)` (NOT `.parse()` ã no throwing)
    - On success: return `{ success: true, data: result.data }`
    - On failure: format Zod issues into user-friendly error array with `path.join('.')`, `message`, `code`

    Implement `parseCalmFromString(jsonString: string): ParseResult`:
    - Try `JSON.parse(jsonString)` first
    - If JSON.parse fails, return ParseError with "Invalid JSON" message
    - If JSON.parse succeeds, delegate to `parseCalm()`

    **Extractor (`src/lib/calm/extractor.ts`):**

    Define `AnalysisInput` interface:
    ```typescript
    interface AnalysisInput {
      nodes: CalmNode[];
      relationships: CalmRelationship[];
      controls: Record<string, ControlDefinition>;
      flows: CalmFlow[];
      metadata: {
        nodeCount: number;
        relationshipCount: number;
        controlCount: number;
        flowCount: number;
        nodeTypes: Record<string, number>;
        relationshipTypes: Record<string, number>;
        protocols: string[];
      };
    }
    ```

    Implement `extractAnalysisInput(calm: CalmDocument): AnalysisInput`:
    - Count nodes by type, relationships by type
    - Extract unique protocols from connect relationships
    - Count controls
    - Return fully structured AnalysisInput

    Implement utility functions:
    - `getNodesByType(calm, nodeType): CalmNode[]`
    - `getNodeRelationships(calm, nodeId): CalmRelationship[]`
    - `getNodeById(calm, nodeId): CalmNode | undefined`

    **IMPORTANT:**
    - All functions are pure (no side effects, no framework imports)
    - Export everything needed by consumers
    - Use barrel export in `src/lib/calm/index.ts` if desired (optional)
  </action>
  <verify>

    - `parseCalm` with invalid JSON returns `{ success: false, error: { issues: [...] } }`
    - `parseCalmFromString` with malformed string returns parse error
    - `extractAnalysisInput` returns AnalysisInput with correct metadata counts
    - All utility functions exported
