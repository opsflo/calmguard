  <name>Task 1: Create GitHub Actions CI workflow with lint, typecheck, test, build, and dependency audit</name>
  <files>.github/workflows/ci.yml, package.json</files>
  <action>
Create `.github/workflows/ci.yml` with these jobs:

**Job: lint** (runs first)
- `ubuntu-latest` runner
- pnpm/action-setup@v4 with version 9
- actions/setup-node@v4 with node-version 22, cache pnpm
- `pnpm install --frozen-lockfile`
- `pnpm lint`
- `pnpm typecheck`

**Job: test** (needs: lint)
- Same pnpm/node setup
- `pnpm install --frozen-lockfile`
- `pnpm test:run` (vitest single run)

**Job: build** (needs: test)
- Same pnpm/node setup
- `pnpm install --frozen-lockfile`
- `pnpm build` with env `NEXT_SKIP_VALIDATE: 1` (build without LLM API keys)

**Job: security** (runs in parallel with lint, no dependencies)
- Same pnpm/node setup
- `pnpm install --frozen-lockfile`
- `pnpm audit --audit-level=high` (block on high+ CVEs only per locked decision: errors only)
- `pnpm license-check` (custom script)

Trigger on: push to main, pull_request to main.

Per locked decision: Block on errors only. Warnings are informational, don't fail.

Add to `package.json` scripts:
- `"license-check"`: `"npx license-checker --failOn 'GPL-2.0;GPL-3.0;AGPL-3.0'"` — check for copyleft licenses (strong signal for financial services judges per locked decision)

Install license-checker as devDependency:
```bash
pnpm add -D license-checker
```

Note: `pnpm audit` may have exit code issues with pnpm v9. Use `pnpm audit --audit-level=high || true` initially if needed, with a comment explaining.
  </action>
  <verify>Validate YAML syntax: `node -e "const yaml = require('yaml'); yaml.parse(require('fs').readFileSync('.github/workflows/ci.yml','utf8'))"` or use `yq` to parse. Verify all 4 jobs exist with correct `needs` dependencies.</verify>
  <done>ci.yml has lint, test, build, and security jobs. Jobs run in correct dependency order. license-check script added to package.json. license-checker installed.</done>

Per locked decision: Both CodeQL AND Semgrep, running as parallel CI jobs (separate workflow files).

Create `.github/workflows/semgrep.yml`:
- Trigger: push to main, PR to main
- Permissions: contents read
- Single job `semgrep`:
  - `ubuntu-latest` runner
  - Container: `semgrep/semgrep`
  - Skip for dependabot: `if: github.actor != 'dependabot[bot]'`
  - `actions/checkout@v4`
  - `semgrep scan --config auto --error` (--error exits non-zero only on error-severity findings, per locked decision: block on errors only)

No Semgrep platform token needed — Community Edition works for open-source with `--config auto`.
