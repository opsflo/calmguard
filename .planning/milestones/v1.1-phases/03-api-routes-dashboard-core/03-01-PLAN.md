      - Error event: `{ type: z.literal('error'), message: z.string() }`

    **src/lib/ai/streaming.ts** (MODIFY):
    Upgrade the singleton to use globalThis pattern to survive webpack hot reloads in Next.js dev mode. Per research pitfall #1, the current module-level `const agentEventEmitter = new AgentEventEmitter()` will break under hot reload because webpack re-instantiates module-level variables.

    Add before the class definition:
    ```typescript
    declare global {
      // eslint-disable-next-line no-var
      var __agentEventEmitter: AgentEventEmitter | undefined;
    }
    ```

    Replace the singleton export:
    ```typescript
    export const agentEventEmitter: AgentEventEmitter =
      globalThis.__agentEventEmitter ??
      (globalThis.__agentEventEmitter = new AgentEventEmitter());
    ```

    Keep the `emitAgentEvent` helper unchanged.

    **src/app/api/analyze/route.ts** (NEW):
    The critical SSE streaming endpoint. Must follow the pattern from research exactly.

    Key implementation:
    1. Add `export const dynamic = 'force-dynamic';` at top to prevent caching
    2. Export `async function POST(req: NextRequest)`
    3. Parse request body using `analyzeRequestSchema`
    4. Parse CALM data: `parseCalm(body.calm)` — return 400 if invalid
    5. Extract analysis input: `extractAnalysisInput(parseResult.data)`
    6. Create ReadableStream:
       - Inside `start(controller)`: subscribe to `agentEventEmitter`
       - Each event: `controller.enqueue(encoder.encode(\`data: ${JSON.stringify(event)}\n\n\`))`
       - Call `runAnalysis(analysisInput)` — this runs agents which emit events
       - On completion: Store pipeline result in globalThis, send `{ type: 'done', result }` event
       - On error: Send `{ type: 'error', message }` event
       - In `finally`: unsubscribe, controller.close()
    7. Return `new Response(stream, { headers: { 'Content-Type': 'text/event-stream; charset=utf-8', 'Cache-Control': 'no-cache, no-transform', 'Connection': 'keep-alive' } })`

    CRITICAL: Do NOT await runAnalysis before returning Response. The Response must be returned synchronously. Agents run inside the ReadableStream start() callback.

    Store the pipeline result on globalThis for the GET /api/pipeline route:
    ```typescript
    declare global {
      var __lastPipelineResult: PipelineConfig | null | undefined;
    }
    ```

    **src/app/api/calm/parse/route.ts** (NEW):
    Simple JSON endpoint for parsing CALM without running agents.

    1. Export `async function POST(req: NextRequest)`
    2. Parse body, call `parseCalm(body.calm)`, call `extractAnalysisInput` on success
    3. Return JSON response with the AnalysisInput structure
    4. Return 400 with error details on parse failure

    **src/app/api/pipeline/route.ts** (NEW):
    Simple GET endpoint that returns the most recent pipeline result.

    1. Export `async function GET()`
    2. Read from `globalThis.__lastPipelineResult`
    3. If exists: return JSON with pipeline data
    4. If null: return JSON with `{ pipeline: null, message: 'No pipeline result available. Run analysis first.' }`
    5. Add `export const dynamic = 'force-dynamic';` to prevent caching
  </action>
