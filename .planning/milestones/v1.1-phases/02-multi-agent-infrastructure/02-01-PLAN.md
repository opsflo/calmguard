<task type="auto">
  <name>Task 1: Create agent type system and SSE event emitter</name>
  <files>
    src/lib/agents/types.ts
    src/lib/ai/streaming.ts
  </files>
  <action>
Create `src/lib/agents/types.ts` with these TypeScript types (all Zod-validated):

1. `AgentEventType` — literal union: `'started' | 'thinking' | 'finding' | 'completed' | 'error'`
2. `Severity` — literal union: `'critical' | 'high' | 'medium' | 'low' | 'info'`
3. `AgentIdentity` — interface with `name: string`, `displayName: string`, `icon: string`, `color: string`
4. `AgentEvent` — interface with:
   - `type: AgentEventType`
   - `agent: AgentIdentity`
   - `message?: string`
   - `severity?: Severity`
   - `data?: unknown`
   - `timestamp: string` (ISO 8601)
5. `AgentConfig` — interface matching YAML structure from research:
   - `apiVersion: string`
   - `kind: 'Agent'`
   - `metadata: { name: string, displayName: string, icon: string, color: string }`
   - `spec: { role: string, model: { provider: string, model: string, temperature: number }, skills: string[], inputs: { type: string }[], outputs: { type: string }[], maxTokens: number }`
6. `AgentResult<T>` — generic interface with `agentName: string`, `success: boolean`, `data?: T`, `error?: string`, `duration: number`
7. Create corresponding Zod schemas with `Schema` suffix for runtime validation (e.g., `agentEventSchema`, `agentConfigSchema`).

Create `src/lib/ai/streaming.ts`:

1. Use Node.js `EventEmitter` pattern (not the `events` module directly — use a typed class).
2. `AgentEventEmitter` class with:
   - `subscribe(listener: (event: AgentEvent) => void): () => void` — returns unsubscribe function
   - `emit(event: AgentEvent): void`
   - Internal `listeners: Set<(event: AgentEvent) => void>` for type safety (avoid Node EventEmitter's `any`)
3. Export singleton `agentEventEmitter` instance.
4. Export `emitAgentEvent(event: Omit<AgentEvent, 'timestamp'>): void` helper that auto-stamps timestamp.
5. No dependency on Node.js `events` module — use simple listener pattern for Edge Runtime compatibility (Vercel serverless).
  </action>
  <verify>Run `pnpm typecheck` — no type errors. Verify both files export expected symbols by checking imports resolve.</verify>
  <done>AgentEvent type system covers all 5 event types with agent identity. Event emitter supports subscribe/emit/unsubscribe with auto-timestamps. No Node.js-specific imports blocking Edge Runtime.</done>
</task>

  <action>
First install AI SDK packages:
```bash
pnpm add ai @ai-sdk/google @ai-sdk/anthropic @ai-sdk/openai @ai-sdk/xai yaml
```

Create `src/lib/ai/provider.ts`:

1. Import provider factories: `google` from `@ai-sdk/google`, `anthropic` from `@ai-sdk/anthropic`, `openai` from `@ai-sdk/openai`, `xai` from `@ai-sdk/xai`.
2. Use `createProviderRegistry` from `ai` package to create a registry with all 4 providers.
3. Only register providers that have API keys set in `process.env`:
   - `GOOGLE_GENERATIVE_AI_API_KEY` → google
   - `ANTHROPIC_API_KEY` → anthropic
   - `OPENAI_API_KEY` → openai
   - `XAI_API_KEY` → xai
4. After conditional registration of all providers, immediately check `Object.keys(providers).length`. If zero, throw `Error('No LLM provider API keys configured. Set at least one of: GOOGLE_GENERATIVE_AI_API_KEY, ANTHROPIC_API_KEY, OPENAI_API_KEY, XAI_API_KEY')`. This check runs at module initialization time in provider.ts (not deferred to first use).
5. Export `registry` — the provider registry instance.
6. Export `getDefaultModel()` that returns `registry.languageModel('google:gemini-2.5-flash')` — fallback logic: try google first, then anthropic (`claude-sonnet-4-20250514`), then openai (`gpt-4o`), then xai (`grok-3`).
7. Export `getModelForAgent(config: AgentConfig)` that resolves `config.spec.model.provider` + `config.spec.model.model` into a language model via registry.
8. Type all exports. No `any`.

NOTE on AI SDK 5 deprecation: Research notes `generateObject` is deprecated in favor of `generateText` with `output` property. Use `generateObject` for now (still works in AI SDK 5.x) — simpler API, migration can happen later. Document this decision in a comment.
  </action>

  <done>Provider registry resolves model strings to actual providers. getDefaultModel() returns Gemini by default with fallback chain. getModelForAgent() resolves from AgentConfig. All 4 providers supported (Google, Anthropic, OpenAI, xAI).</done>
