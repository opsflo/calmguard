**Architecture Analyzer** (`src/lib/agents/architecture-analyzer.ts`):

1. Define `architectureAnalysisSchema` (Zod) at top of file:
   ```
   {
     components: [{
       nodeId: string,
       name: string,
       type: NodeType,
       description: string,
       securityControls: string[],
       dataClassification: string (optional)
     }],
     dataFlows: [{
       sourceId: string,
       destinationId: string,
       protocol: string (optional),
       description: string,
       dataTypes: string[],
       encrypted: boolean
     }],
     trustBoundaries: [{
       name: string,
       description: string,
       nodeIds: string[],
       boundaryType: 'network' | 'security-zone' | 'deployment' | 'organizational'
     }],
     securityZones: [{
       name: string,
       nodes: string[],
       trustLevel: 'high' | 'medium' | 'low' | 'untrusted'
     }],
     findings: [{
       finding: string,
       severity: Severity,
       affectedNodes: string[],
       recommendation: string
     }],
     summary: string
   }
   ```
2. Export type: `type ArchitectureAnalysis = z.infer<typeof architectureAnalysisSchema>`
3. Implement `analyzeArchitecture(input: AnalysisInput): Promise<AgentResult<ArchitectureAnalysis>>`:
   - Load agent config: `loadAgentConfig('architecture-analyzer')`. Use `config.metadata` to construct `AgentIdentity` for all emitted events: `{ name: config.metadata.name, displayName: config.metadata.displayName, icon: config.metadata.icon, color: config.metadata.color }`.
   - Get model: `getModelForAgent(config)` (fallback to `getDefaultModel()`)
   - Emit `started` event with agent identity constructed from config.metadata above
   - Build prompt: Include role from config, serialize `input` as JSON, instruct to analyze components, data flows, trust boundaries
   - Emit `thinking` event: "Analyzing architecture components and data flows..."
   - Call `generateObject({ model, schema: architectureAnalysisSchema, prompt })`
   - Wrap in retry logic: 3 attempts with exponential backoff (1s, 2s, 4s)
   - Emit `finding` events for each finding in result (with severity)
   - Emit `completed` event
   - Return `AgentResult<ArchitectureAnalysis>` with `data`, `success: true`, `duration` (track start time with `performance.now()`)
   - On error: emit `error` event, return `AgentResult` with `success: false`, `error` message

**Compliance Mapper** (`src/lib/agents/compliance-mapper.ts`):

1. Define `complianceMappingSchema` (Zod):
   ```
   {
     frameworkMappings: [{
       framework: 'SOX' | 'PCI-DSS' | 'CCC' | 'NIST-CSF',
       controlId: string,
       controlName: string,
       calmControlId: string (optional — null if gap),
       status: 'compliant' | 'partial' | 'non-compliant' | 'not-applicable',
       evidence: string,
       recommendation: string,
       severity: Severity
     }],
     frameworkScores: [{
       framework: 'SOX' | 'PCI-DSS' | 'CCC' | 'NIST-CSF',
       score: number (0-100),
       totalControls: number,
       compliantControls: number,
       partialControls: number,
       nonCompliantControls: number
     }],
     gaps: [{
       framework: string,
       missingControl: string,
       description: string,
       severity: Severity,
       recommendation: string
     }],
     summary: string
   }
   ```
2. Export type: `type ComplianceMapping = z.infer<typeof complianceMappingSchema>`
3. Implement `mapCompliance(input: AnalysisInput): Promise<AgentResult<ComplianceMapping>>`:
   - Load agent config: `loadAgentConfig('compliance-mapper')`. Use `config.metadata` to construct `AgentIdentity` for all emitted events (same pattern as Architecture Analyzer: `{ name, displayName, icon, color }` from config.metadata).
   - Load skills: `loadSkillsForAgent(config)` — this injects SOX, PCI-DSS, CCC, NIST-CSF knowledge
   - Get model, emit `started` with AgentIdentity from config.metadata, build prompt with skills content + input JSON
   - Emit `thinking`: "Mapping controls to SOX, PCI-DSS, CCC, and NIST-CSF frameworks..."
   - Call `generateObject` with retry logic
   - Emit `finding` events for non-compliant and partial mappings
   - Emit `completed`, return result

Same retry pattern, error handling, and AgentResult wrapper as Architecture Analyzer.

**Risk Scorer** (`src/lib/agents/risk-scorer.ts`):

1. Define `riskAssessmentSchema` (Zod):
   ```
   {
     overallScore: number (0-100),
     overallRating: 'critical' | 'high' | 'medium' | 'low',
     frameworkScores: [{
       framework: 'SOX' | 'PCI-DSS' | 'CCC' | 'NIST-CSF',
       score: number (0-100),
       rating: 'critical' | 'high' | 'medium' | 'low'
     }],
     nodeRiskMap: [{
       nodeId: string,
       nodeName: string,
       riskLevel: 'critical' | 'high' | 'medium' | 'low',
       riskFactors: string[],
       complianceGaps: number
     }],
     topFindings: [{
       finding: string,
       severity: Severity,
       affectedNodes: string[],
       framework: string (optional),
       recommendation: string
     }],
     summary: string,
     executiveSummary: string
   }
   ```
2. Export type: `type RiskAssessment = z.infer<typeof riskAssessmentSchema>`
3. Define and export `RiskScorerInput` interface (not AnalysisInput — this agent takes aggregated results):
   ```
   {
     architecture: ArchitectureAnalysis,
     compliance: ComplianceMapping,
     pipeline: PipelineConfig,
     originalInput: AnalysisInput  // REQUIRED: the full CALM input for context — passed by orchestrator
   }
   ```
   Import `AnalysisInput` from `@/lib/calm/extractor`. The orchestrator (Plan 02-05) is responsible for passing the original CALM input as `originalInput` when calling `scoreRisk()`.
4. Implement `scoreRisk(input: RiskScorerInput): Promise<AgentResult<RiskAssessment>>`:
   - Load config, get model, emit events
   - Build prompt: Include all 3 agent results as JSON, instruct to aggregate into overall score, per-framework scores, and node risk map
   - Emit `thinking`: "Aggregating results and calculating risk scores..."
   - Call `generateObject` with retry
   - Emit `finding` events for top findings (critical and high severity)
   - Emit `completed`
   - Return result

All agents share the same pattern:
- Load config → get model → emit started → build prompt → emit thinking → generateObject with retry → emit findings → emit completed → return AgentResult
- Error handling: try/catch, emit error event, return `{ success: false, error: message, duration }`
- Retry: 3 attempts, exponential backoff (1s, 2s, 4s), append error feedback to prompt on retry

Extract a shared helper `createAgentRunner` or just repeat the pattern (4 agents is manageable without abstraction overhead).
