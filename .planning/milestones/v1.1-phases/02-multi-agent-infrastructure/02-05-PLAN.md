<task type="auto">
  <name>Task 1: Implement orchestrator with parallel/sequential execution</name>
  <files>
    src/lib/agents/orchestrator.ts
  </files>
  <action>
Create `src/lib/agents/orchestrator.ts`:

1. Import all 4 agent functions and their result types:
   - `analyzeArchitecture` + `ArchitectureAnalysis` from `./architecture-analyzer`
   - `mapCompliance` + `ComplianceMapping` from `./compliance-mapper`
   - `generatePipeline` + `PipelineConfig` from `./pipeline-generator`
   - `scoreRisk` + `RiskAssessment` from `./risk-scorer`

2. Import `emitAgentEvent` from `@/lib/ai/streaming`, `loadAgentConfig` from `./registry`, `AnalysisInput` from `@/lib/calm/extractor`. The orchestrator loads its own config via `loadAgentConfig('orchestrator')` and constructs its `AgentIdentity` from `config.metadata` (name, displayName, icon, color) for all orchestrator-level events (phase-started, phase-complete, agent failure warnings).

3. Define `AnalysisResult` interface and Zod schema:
   ```
   {
     architecture: ArchitectureAnalysis | null,
     compliance: ComplianceMapping | null,
     pipeline: PipelineConfig | null,
     risk: RiskAssessment | null,
     duration: number,
     completedAgents: string[],
     failedAgents: string[]
   }
   ```
   Use `| null` for each agent result so partial results work if an agent fails (graceful degradation per INFRA-05).

4. Implement `runAnalysis(input: AnalysisInput): Promise<AnalysisResult>`:

   a. Load orchestrator config: `loadAgentConfig('orchestrator')`
   b. Emit orchestrator `started` event
   c. Track start time with `performance.now()`

   d. **Phase 1 — Parallel execution** (3 agents):
      - Emit thinking: "Running Architecture Analyzer, Compliance Mapper, and Pipeline Generator in parallel..."
      - Use `Promise.allSettled([analyzeArchitecture(input), mapCompliance(input), generatePipeline(input)])`
      - `allSettled` (NOT `all`) — so one agent failure doesn't cancel others
      - Extract results: for each settled promise, if fulfilled AND `result.success`, keep data. Otherwise set to null and add to `failedAgents`.
      - Emit orchestrator finding if any agent failed: "Agent X failed: {error}"

   e. **Phase 2 — Sequential execution** (Risk Scorer):
      - Only run if at least architecture AND compliance succeeded (risk scoring needs both)
      - If skipping: emit warning event, set risk to null
      - Emit thinking: "Running Risk Scorer with aggregated results..."
      - Call `scoreRisk({ architecture, compliance, pipeline, originalInput: input })`
      - Handle failure: set risk to null, add to failedAgents

   f. Emit orchestrator `completed` event with summary (e.g., "Analysis complete: 4/4 agents succeeded" or "3/4 agents succeeded")

   g. Return `AnalysisResult` with all agent outputs, duration, completed/failed agent lists

5. Error handling: Wrap entire function in try/catch. On catastrophic error (e.g., no providers configured), emit `error` event and throw.

6. Export: `runAnalysis`, `AnalysisResult`, `analysisResultSchema`
  </action>
  <verify>Run `pnpm typecheck`. Orchestrator imports all 4 agents and their types. Promise.allSettled is used for parallel execution. AnalysisResult supports null for failed agents.</verify>
  <done>Orchestrator runs 3 agents in parallel (Phase 1) then Risk Scorer sequentially (Phase 2). Uses Promise.allSettled for graceful degradation. Emits orchestrator-level events. Returns combined AnalysisResult with partial result support.</done>
