Create `skills/` directory at project root. Create 4 markdown files with substantive compliance framework knowledge that will be injected into the Compliance Mapper agent's prompt.

Each SKILL.md should contain:
- **Framework overview** (what it is, who it applies to)
- **Key control areas** (major sections/domains)
- **Specific controls relevant to CALM architectures** (data encryption, access control, audit logging, network segmentation, etc.)
- **Mapping guidance** (how to map CALM controls to this framework's requirements)
- **Common findings** (typical non-compliance patterns in financial services architectures)

1. **skills/SOX.md** — Sarbanes-Oxley Act compliance:
   - Section 302 (Corporate Responsibility for Financial Reports)
   - Section 404 (Management Assessment of Internal Controls)
   - IT General Controls (ITGC): access controls, change management, operations, SDLC
   - Relevant to CALM: audit logging controls, data integrity, access control patterns
   - Mapping: CALM `audit-logging` → SOX ITGC, CALM `data-encryption` → SOX data integrity

2. **skills/PCI-DSS.md** — Payment Card Industry Data Security Standard v4.0:
   - 12 Requirements organized in 6 goals
   - Key requirements: Req 1 (firewalls), Req 3 (protect stored data), Req 4 (encrypt transmission), Req 6 (secure systems), Req 8 (authentication), Req 10 (monitor access), Req 11 (testing)
   - Relevant to CALM: network segmentation, encryption controls, authentication patterns
   - Mapping: CALM `data-encryption` → PCI-DSS Req 3/4, CALM protocol `mTLS` → PCI-DSS Req 4

3. **skills/FINOS-CCC.md** — FINOS Common Cloud Controls:
   - Cloud-agnostic controls for financial services
   - Categories: Identity & Access, Data Protection, Network Security, Logging & Monitoring, Resilience, Compliance
   - Relevant to CALM: deployment controls, cloud infrastructure patterns
   - Mapping: CALM `deployed-in` relationships → CCC infrastructure controls

4. **skills/NIST-CSF.md** — NIST Cybersecurity Framework 2.0:
   - 6 Functions: Govern, Identify, Protect, Detect, Respond, Recover
   - Categories within each function relevant to software architecture
   - Relevant to CALM: all control types map to NIST CSF subcategories
   - Mapping: CALM controls → NIST CSF subcategories (e.g., `data-encryption` → PR.DS-1)

Each file should be 100-200 lines of substantive content — enough to meaningfully inform the LLM but not so long it wastes tokens. Focus on controls that can be detected in CALM architectures (nodes, relationships, protocols, controls objects).
