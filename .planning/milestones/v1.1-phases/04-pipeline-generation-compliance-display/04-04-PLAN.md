       - Add `toggleFramework: (framework: string) => void` action that prevents removing the last framework (if length === 1 and framework matches, return unchanged state)
       - Add both to initialState and store implementation
       - Do NOT reset selectedFrameworks in `startAnalysis` or `reset` — user's framework selection should persist across analyses

    3. Update `src/lib/api/schemas.ts`:
       - Add optional `frameworks` field to `analyzeRequestSchema`:
         ```typescript
         frameworks: z.array(z.enum(['SOX', 'PCI-DSS', 'CCC', 'NIST-CSF'])).optional()
         ```

    4. Update `src/hooks/use-agent-stream.ts`:
       - Change `startStream` signature to accept `(calmData: unknown, selectedFrameworks?: string[])`
       - Include `frameworks: selectedFrameworks` in the POST body JSON.stringify alongside `calm: calmData`
       - Import `toast` from `sonner` and call `toast.error('Analysis failed', { description: errorMessage })` in all error paths:
         - HTTP error path (after setting status to error)
         - Max retries exhausted path
         - Null response.body path

    5. Update `src/app/api/analyze/route.ts`:
       - Extract `frameworks` from `bodyResult.data.frameworks`
       - Pass as second argument to `runAnalysis(analysisInput, frameworks)`

    6. Update `src/lib/agents/orchestrator.ts`:
       - Change `runAnalysis` signature to `(input: AnalysisInput, selectedFrameworks?: string[])`
       - Pass `selectedFrameworks` to `mapCompliance(input, selectedFrameworks)` and `scoreRisk({...}, selectedFrameworks)` — these agents use it in their prompts to filter analysis scope
       - Update the compliance mapper call: `mapCompliance(input, selectedFrameworks)` — the compliance mapper already accepts a second parameter or will need its function signature updated to accept and use it
       - Update the risk scorer call: pass selectedFrameworks as part of the options object
       - NOTE: If `mapCompliance` and `scoreRisk` don't accept frameworks yet, add the parameter to their signatures but don't change their internal logic — they'll just ignore unused parameters until a future plan. The key change is that the parameter flows through.

    7. Update `src/components/dashboard/header.tsx`:
       - Read `selectedFrameworks` from `useAnalysisStore`:
         ```typescript
         const selectedFrameworks = useAnalysisStore((state) => state.selectedFrameworks);
         ```
       - Update `handleAnalyze` to pass selectedFrameworks to startStream:
         ```typescript
         const handleAnalyze = () => {
           if (rawCalmData) {
             void startStream(rawCalmData, selectedFrameworks);
           }
         };
         ```
       - This is the PRIMARY "Analyze" button in the dashboard header — it MUST pass selectedFrameworks to startStream, otherwise the framework selector on the landing page has no effect on the main analysis trigger.
